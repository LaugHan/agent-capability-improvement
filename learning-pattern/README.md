# 📚 判例式学习

> 来源：L站社区综合洞见
> 启发：像法律判例一样积累经验

## 核心思想

不是学习"规则"，而是学习"案例"。

```
规则式学习:
"遇到错误要记录" → 死板，可能不适用

判例式学习:
"上次类似情况这样做成功了" → 灵活、实用
```

## 判例结构

每个案例记录：

```markdown
## 案例: [简短标题]

### 情况
- 什么场景？

### 行动
- 做了什么？

### 结果
- 成功/失败？

### 教训
- 核心洞见是什么？

### 可复用条件
- 什么情况下可以参考这个案例？
```

## 判例库示例

```python
# cases/vague-question.yaml
- id: vague-question-001
  situation: "用户说'帮我弄一下'，没有具体说明"
  action: "询问具体需求，给出选项"
  result: "成功 - 用户澄清了需求"
  lesson: "不明确时先澄清，不要猜测"
  tags: [communication, clarification]

# cases/error-recovery.yaml  
- id: error-recovery-001
  situation: "工具调用失败"
  action: "记录错误，尝试替代方案"
  result: "成功 - 3次尝试后完成"
  lesson: "错误是信息，不是失败"
  tags: [error-handling, persistence]
```

## 社区学习策略

从高质量帖子学习：

1. **看标题**: 快速判断是否相关
2. **扫结构**: 找核心洞见
3. **挖细节**: 理解为什么这样做
4. **想迁移**: 这个洞见能用到哪？
