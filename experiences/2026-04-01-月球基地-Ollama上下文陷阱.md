# OpenClaw + Ollama 上下文窗口陷阱

来源: 月球基地
作者: -
链接: https://blog.eimoon.com/p/openclaw-ollama-local-llm-guide/
日期: 2026-04-01

## 核心

⚠️ **这是本地部署 OpenClaw 最容易踩的坑！**

### 问题

Ollama 默认 `OLLAMA_NUM_CTX=2048`，意思是"上下文只能存 2048 个 token"。

但 OpenClaw 代理最少需要 **16K-24K** token 才能正常工作，因为：
- 系统提示词 (system prompts)
- 工具定义
- 对话历史
- 工具调用结果

这些加起来，一个中等复杂度的单次交互就轻松消耗 8,000-12,000 token。

### 症状

- 交互式手动测试：✅ 正常
- 设置定时任务自动跑：❌ 输出垃圾内容/语无伦次
- 没有报错，只是"看起来不对"

**原因**：Ollama 静默截断超出 2048 token 的内容，模型只看到实际对话的 10%，然后基于残缺信息回复。

### 解决

启动 Ollama 前设置环境变量：

```bash
export OLLAMA_NUM_CTX=24576
```

24576 ≈ 24K，正好满足 OpenClaw 的需求，还有余量给工具定义。

## 启发

本地部署 Agent 时，基础配置往往比"模型选型"更重要：
- 上下文窗口是硬性约束
- 默认值 ≠ 适合 Agent 的值
- 调参之前先检查基础配置

调试"不对劲但没报错"的问题时，先检查：
1. 上下文窗口够不够
2. 环境变量有没有生效
3. Docker 有没有正确重启

## 实际用法

1. **首次部署 OpenClaw + Ollama**：第一件事就是设 `OLLAMA_NUM_CTX=24576`
2. **遇到定时任务输出异常**：先检查上下文窗口
3. **调试任何"静默失败"**：看环境变量有没有正确加载
