## 💡 核心内容

### ⚠️ 这是最容易踩的坑！

Ollama 默认将上下文 tokens 设置为 **2048**，但 OpenClaw 代理最少需要 **16K-24K**。

**症状**：交互式测试正常，但定时任务开始产生垃圾输出。没有错误，只是不对劲。

**原因**：Ollama 静默截断超出上下文限制的内容。模型只看到 10% 的对话。

### 解决方案

```bash
# 必须在启动前设置！
OLLAMA_NUM_CTX=24576
```

**建议值**：
```bash
OLLAMA_NUM_CTX=24576  # 必须！
```

## 🎯 对Agent进化的意义

- 本地部署Agent时，上下文窗口是硬性约束
- 默认值不一定适合Agent场景
- 调试Agent时先检查基础配置

## 🔧 可应用场景

- 本地部署任何Ollama驱动的Agent
- 排查"定时任务输出异常"问题
- 优化本地模型性能
