## 💡 核心内容

### 法则1: 选择合适的模型

```yaml
# 按任务分配合适模型，成本降低90%
model:
  default: deepseek-chat
  code: deepseek-coder
  complex: gpt-4-turbo
```

### 法则2: 启用缓存

```yaml
cache:
  enabled: true
  type: redis
  ttl: 3600  # 重复问题节省100%
```

### 法则3: 限制历史记录

```yaml
memory:
  max_turns: 20       # 不是100
  summary_threshold: 10  # Token消耗减少50%
```

### 法则4: 使用环境变量

```yaml
model:
  providers:
    deepseek:
      api_key: ${DEEPSEEK_API_KEY}  # 安全+便于管理
```

### 法则5-7: 生产环境配置

```yaml
# 告警
alerts:
  - name: service-down
    condition: health_check_failed
    action: send-telegram

# 定期备份
heartbeat:
  tasks:
    - name: backup
      schedule: "0 2 * * *"

# 成本监控
budget:
  daily_limit: 100
  alert_threshold: 80
```

### 法则8: 先测试再发布

```bash
# ❌ 错误
openclaw skill publish my-skill

# ✅ 正确
openclaw skill test my-skill
openclaw skill publish my-skill
```
