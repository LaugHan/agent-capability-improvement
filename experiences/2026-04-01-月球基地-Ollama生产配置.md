## 核心

本地部署 Ollama + OpenClaw 到生产环境，不能只靠 `ollama serve`。

### 完整环境变量

```bash
OLLAMA_HOST=0.0.0.0              # 允许外部访问
OLLAMA_KEEP_ALIVE=1h             # 模型保留1小时（默认5分钟太短）
OLLAMA_NUM_CTX=24576             # 上下文窗口（必须！详见上下文陷阱）
OLLAMA_FLASH_ATTENTION=1         # 启用Flash Attention加速
OLLAMA_KV_CACHE_TYPE=q8_0        # 缓存量化，省50%显存
OLLAMA_NUM_PARALLEL=2            # 并发2个请求
CUDA_VISIBLE_DEVICES=0           # 指定GPU编号
```

### 参数详解

| 参数 | 作用 | 默认值 | 生产建议 |
|------|------|--------|----------|
| `OLLAMA_NUM_CTX` | 上下文窗口 | 2048 | 24576（必须改） |
| `OLLAMA_FLASH_ATTENTION` | 加速推理 | 0 | 1 |
| `OLLAMA_KV_CACHE_TYPE` | 显存优化 | - | q8_0（省50%显存） |
| `OLLAMA_NUM_PARALLEL` | 并发数 | 1 | 2（看显存） |
| `OLLAMA_KEEP_ALIVE` | 模型保留时间 | 5m | 1h（避免冷启动） |
| `CUDA_VISIBLE_DEVICES` | GPU分配 | 全部 | 0（专用） |

### 推荐模型

| 模型 | 适用场景 | 显存需求 | 说明 |
|------|----------|----------|------|
| `qwen3:30b-a3b` | 主选 | ~21GB | MoE架构，30B参数只激活3B |
| `qwen2.5:14b` | 中端 | ~12GB | RTX 3090可跑 |
| `qwen3:0.6b` | 轻量 | - | 仅预处理/嵌入 |

**注意**：工具调用是必要条件，没有它 Agent 无法执行操作，只能"说"不能"做"。

### Docker 陷阱

```bash
# ❌ 错误 - 不会应用新的环境变量
docker restart openclaw

# ✅ 正确 - 完整重建容器
docker compose down && docker compose up -d
```

修改 `docker-compose.yml` 或环境变量后，必须完整重建。

### 多 GPU 分配

如果机器上有多块 GPU：

```bash
# Ollama 专用 GPU 0
CUDA_VISIBLE_DEVICES=0

# 避免和其他 CUDA 服务共享，导致显存冲突崩溃
```

## 启发

本地部署"看起来简单"，但有非常多的坑：
1. 上下文窗口默认值太小
2. Docker 修改环境变量不会自动生效
3. 多服务共享 GPU 会导致不稳定

生产环境 = 完整配置 + 监控 + 备份，不是"能跑就行"。

## 实际用法

1. **首次部署**：按上表设置所有环境变量
2. **显存不够**：加 `OLLAMA_KV_CACHE_TYPE=q8_0`
3. **请求排队慢**：适当增加 `OLLAMA_NUM_PARALLEL`
4. **修改配置后不生效**：用 `docker compose down && up`
