## 💡 核心内容

### 完整环境变量

```bash
OLLAMA_HOST=0.0.0.0
OLLAMA_KEEP_ALIVE=1h
OLLAMA_NUM_CTX=24576
OLLAMA_FLASH_ATTENTION=1
OLLAMA_KV_CACHE_TYPE=q8_0
OLLAMA_NUM_PARALLEL=2
NVIDIA_VISIBLE_DEVICES=all
CUDA_VISIBLE_DEVICES=0
```

### 参数说明

| 参数 | 作用 | 推荐值 | 说明 |
|------|------|--------|------|
| `OLLAMA_NUM_CTX` | 上下文窗口 | 24576 | 必须！默认2048太小 |
| `OLLAMA_FLASH_ATTENTION` | 加速推理 | 1 | 启用Flash Attention |
| `OLLAMA_KV_CACHE_TYPE` | 缓存量化 | q8_0 | 省50%显存 |
| `OLLAMA_NUM_PARALLEL` | 并发请求 | 2 | 多Agent共享 |
| `OLLAMA_KEEP_ALIVE` | 模型保留时间 | 1h | 默认5分钟太短 |
| `CUDA_VISIBLE_DEVICES` | 指定GPU | 0 | 专用GPU |

### 推荐模型

| 模型 | 说明 | 显存 |
|------|------|------|
| `qwen3:30b-a3b` | 首选，MoE，30B参数激活3B | ~21GB |
| `qwen2.5:14b` | 中端替代 | ~12GB |
| `qwen3:0.6b` | 轻量预处理 | - |

### Docker 重启问题

```bash
# ❌ 错误 - 不会应用环境变量更改
docker restart openclaw

# ✅ 正确
docker compose down && docker compose up -d
```

## 🎯 对Agent进化的意义

- 生产环境需要完整配置，不能只靠默认
- 显存优化让消费级GPU也能跑Agent
- Docker配置变更需要完整重启

## 🔧 可应用场景

- 本地部署Agent到生产环境
- 优化GPU资源利用率
- 配置多Agent并发
