## 核心

> "与其增加容量，不如优化触发"

### 两种优化方向对比

**容量优化思维**（常见但低效）：
- 遇到问题 → "我知识库不够，加更多"
- 模型慢了 → "扩展上下文，加长"
- 效果不好 → "加载更多文件，增大知识面"

**触发优化思维**（高效）：
- 遇到问题 → "什么情况应该激活什么知识？"
- 模型慢了 → "是不是不需要的也加载了？"
- 效果不好 → "触发的时机和条件对吗？"

### 本质区别

```
容量优化 = 堆料思维
触发优化 = 精准思维
```

不是"有什么"，而是"什么时候用"。

### 简单实现

```python
# 不是把所有工具都塞给 Agent
all_tools = [tool1, tool2, tool3, tool4, tool5, ...]

# 而是根据情况触发相关工具
triggers = {
    "code_error": [debug_tools, search_memory, ask_clarification],
    "vague_request": [clarify_tools, example_discovery],
    "long_context": [summarize_tools, buffer_management],
    "file_operation": [read_file, write_file, exec]
}

def activate_tools(context):
    """根据上下文类型，精准触发工具"""
    tool_types = detect_context_type(context)
    return [t for t in all_tools if t.type in tool_types]
```

## 启发

Agent 常犯的错：
- 加载太多无关信息，反而干扰
- 什么工具都暴露，分散注意力
- 上下文太长，主要信息被淹没

触发优化的核心是：**在正确的时间、给正确的信息、激活正确的工具**

## 实际用法

1. **设计 Agent 系统时**：先想"什么情况激活什么"，而不是"我要做多大"
2. **上下文快满时**：不是扩展长度，而是筛选相关性
3. **工具调用时**：根据当前任务类型，决定暴露哪些工具
4. **知识检索时**：先匹配触发条件，而不是向量搜索全量

关键问题变成：
- "这个信息什么时候会被用到？"
- "这个工具在什么情况下需要？"
- "这条知识触发的条件是什么？"
