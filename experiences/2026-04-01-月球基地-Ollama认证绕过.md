## 💡 核心内容

### 问题

OpenClaw Gateway 要求每个 Provider 有 API Key，但 Ollama 不需要 API Key。

如果尝试设置 `type: "none"`，会在热重载时被剥离。

### 解决方案

在 provider 配置中设置一个**虚假的 apiKey 值**，并配合 `authHeader: false`：

```yaml
providers:
  ollama:
    apiKey: "dummy-key"
    authHeader: false
```

**注意**：这个配置很少有人记录，没有它系统会静默阻止代理运行。

## 🎯 对Agent进化的意义

- 某些系统有隐性约束，需要变通
- "dummy" 值可以绕过不必要的验证
- 文档不一定记录所有边界情况
