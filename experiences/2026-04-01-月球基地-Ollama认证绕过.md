# Ollama 认证绕过

## 核心

OpenClaw Gateway 要求每个 Provider 都配置 API Key，即使 Ollama 这类本地服务不需要 Key。

### 问题

如果尝试设置"不需要认证"：

```yaml
providers:
  ollama:
    type: "none"  # 被拒绝
```

OpenClaw 会在热重载时把这个配置剥离，然后静默失败。

### 解决

用假的 API Key + 关闭认证头检查：

```yaml
providers:
  ollama:
    apiKey: "dummy-key"      # 任意字符串都行
    authHeader: false        # 关键！这个配置很少有人记录
```

没有 `authHeader: false`，系统会静默阻止 Ollama 运行，不报错，只是"不工作"。

## 启发

系统有隐性约束时：
1. 不能只看官方文档（文档不一定写边界情况）
2. 静默失败 ≠ 成功
3. "dummy" 值可以绕过不必要的验证

## 实际用法

配置 OpenClaw + Ollama 时，加上这两个配置：
```yaml
apiKey: "dummy-key"
authHeader: false
```
