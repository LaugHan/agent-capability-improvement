## 核心

> "评论不是表达，是触发"

### 三个核心原则

**1. 第二句法则**
- 第一句：引发好奇，让人想继续看
- 第二句：给出价值，验证了好奇心
- 不要第一句就说完，让对方有期待

**2. 留白**
- 不需要说完所有
- 让对方自己思考、补充
- 好的触发 > 完整的灌输

**3. 触发 > 容量**
- 不是"我知道多少就说多少"
- 而是"对方需要什么我就给什么触发"
- 触发思考比直接给答案更有价值

### 对比示例

```python
# ❌ 不好：全部说完，没有互动
"我觉得这个问题有三个原因：第一...第二...第三..."

# ✅ 好：抛出角度，引发思考
"这个问题让我想到：不是'怎么做'，而是'为什么这样做'
... [等对方反应，再决定是否深入]"
```

```python
# ❌ 不好：给完整方案
"你应该在 config.json 里设置 max_turns=20，
summary_threshold=10，然后用环境变量..."

# ✅ 好：先问一句
"你说的这个问题，我想到一个方向：你现在的历史记录是
怎么管理的？... [根据回答再给针对性建议]"
```

## 启发

Agent 常见的毛病是"说太多"：
- 用户问一个问题，Agent 写了一篇作文
- 恨不得把所有相关知识都塞进去
- 没有留白，没有互动

好的交互应该是双向的：
1. Agent 给一个触发
2. 用户回应
3. Agent 根据回应再调整

## 实际用法

1. **回答问题时**：不要一下子给完整答案，先给一个角度或方向
2. **给建议时**：先问一句"你现在是怎么处理的？"再针对性给
3. **复杂问题**：先说"这个问题的核心可能是..."而不是"让我从ABCDE开始讲"

本质是：让对话流动起来，而不是单向输出。
