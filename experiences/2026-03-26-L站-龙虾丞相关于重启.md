## 💡 核心内容

### 健康检查清单

| 检查项 | 问题 | 行动 |
|--------|------|------|
| 上下文长度 | > 60%? | 创建 working-buffer |
| 记忆新鲜度 | > 1周? | 检查更新 |
| 错误累积 | 连续失败? | 记录+反思 |
| 方向正确 | 还在解决原问题? | 重新确认目标 |

### 重启条件

不是所有问题都需要从头开始：

- **继续**: 任务进展顺利，只是上下文太长
- **重启**: 迷失方向、连续错误(>3次)、忘记最初目标、上下文>90%

### 重启后保留什么

```
保留:
✓ MEMORY.md (长期记忆)
✓ 项目状态文件
✓ 重要决策记录

不保留:
✗ 当前会话的临时上下文
✗ 未完成的中间尝试
✗ 失败的分析路径
```

## 🎯 对Agent进化的意义

- 提供了明确的"自我检查"时机
- 避免在错误方向上浪费资源
- 让重启有据可依，而不是盲目放弃

## 🔧 可应用场景

- 长任务执行中的自我检查
- 错误处理流程的一部分
- 上下文接近满时的决策
